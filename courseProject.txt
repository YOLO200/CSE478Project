Course Project
The course project will cover the last ~8 weeks of the course. Students will work in self-selected groups of 3-5. The completed project will require you to showcase technical proficiency in web and D3 programming as well as the ability to effectively communicate data insights using visualization design concepts and theory from the course lectures.

For the course project, you will create a visual data story that follows (most of) the requirements for the PacificVis Storytelling Contest.

Technical Requirements
The PacificVis Storytelling Contest (https://visstory.github.io/Links to an external site.) is a yearly competition that solicits interesting and expressive data-driven narratives based around the creation and use of visualizations. If you choose this option, you will create your own story that would be suitable for submitting to the 2025 contest, according to the requirements outlined on the contest site. (Note: You're not actually submitting this to the contest; you'll just use the contest rules for your project.)

Before selecting this option, I highly recommend you review prior contest entries! They will give you a good idea of the different types of topics/stories that are told, the types of datasets that are used, and the types of visualization techniques and storytelling mechanisms that are applied. There are many ways to tell a data story, as outlined on the contest site. Here's some common examples:

Create a large infographic that combines visualizations and visual design.
Create and showcase a series of visualizations via a video
Create a scrollytelling story that combines visualizations and other forms of media with narrative text
Create a storytelling website or dashboard
and more!...
Note that the contest site shows videos for several entries, though the entries themselves were not first implemented as videos. 

The contest site lists many requirements (e.g., "The dataset(s) used in the story must be publicly available, and linked to, sourced, or otherwise referenced.") that you should follow. There are also some additional requirements specifically for CSE 478 that you must follow:

You must include at least 5 different visualization techniques in the entry.
Note: 3 bar charts == 1 visualization technique
Variants of a visualization count as the same technique.  For example, the following are all under the "bar chart" umbrella: bar chart, stacked bar chart, radial bar chart, radar chart, diverging bar chart, normalized bar chart, grouped bar chart, bar chart with annotations or glyphs on top, isotype chart, and so on. Likewise, a "line chart" would include multi-line chart, area chart, stacked area chart, streamgraph, etc.
One of these visualizations must be innovative/novel (we'll discuss what this means during lecture)
You cannot have a "static" submission, such as a non-interactive image/PDF. As an example, if you want to implement an infographic type of of contest entry, you would need to make at least part of it interactive based on a user's actions or interactions (alternatively, you could create an animated visualization entry or movie, which although it might not support user interactions, would still not be static and thus fulfill this requirement).
Your implementation must be done using D3.js (or if you are using 3D charts, libraries like three.js are allowed).
The goal should be to create an interesting and story, both in terms of narrative and visualization designs/interactions. We will expect high-quality implementations that are on par with the types of entries presented on the contest site.

Regarding the datasets for your project:
Some groups will create their entire story using 1 dataset. Other groups will pick a different dataset for each visualization in their story. Either option is fine, but remember: you are creating a story, so there must be a narrative through the visualizations/datasets that you use!
These project topics are not allowed:
COVID
Video Games
Kaggle and CORGIS are two sites that have a lot of open datasets that you might find good to use. You are also allowed to "hand-create" datasets, though this can take a lot of work (and you will be counted off for sloppy work).
A "fake" dataset that you make up on your own is NOT allowed.
An example of this would be something like, "We want to create some fake reviews for movies, so we used ChatGPT to make 1,000 reviews and visualized the result."
A real-world model that outputs synthetic/modeled data is allowed though. Example: "We used the XYZ machine learning model to simulate temperature change over the next 100 years and visualized the result."
Particularly if you're using 1 dataset only, I recommend that you pick datasets that are big and have lots of attributes (or data points). Datasets that are "too simple" is sometimes difficult to properly tell a story.
You can pre-process your datasets if you want, where each visualization accesses only the necessary data that it needs, or you can do data transforms using D3/Javascript code.
Your story should be as self-explanatory as possible.
Employ good design (axes, labels, annotations, consistent styling and theming, etc.)
Most teams will implement their story as a web page. If you're doing something different (eg, a video), discuss with Dr. Bryan.
Your visualizations should be created using D3.
CSS frameworks such as Bootstrap and JavaScript libraries such as jQuery, leaflet.s, and moment.js are allowed, but you must use D3 to implement your charts. Libraries that "simplify" D3, such as NVD3, Vega-Lite, Highcharts, etc., are not allowed.
The "innovative view"
As you might have read above, one of your charts is supposed to be an "innovative view." This view should either be a custom-designed visualization (ie, marks and channels custom-written using D3) or it should be an extension of an existing technique (e.g., adding new channels and layers onto an existing chart). In previous years, many students have fulfilled this requirement by combining two types of charts together.

The innovative view is worth 5% of your total project grade. Put another way, your team has the option to simply submit your project containing "normal," non-innovative visualizations. If this happens, you can still receive an overall excellent score of 95/100 on the project.

Goal and Grading
The goal of the course project is to synthesize knowledge both from visualization design and theory (lectures, readings, etc.) with D3 programming. Projects will be evaluated using the following high-level criteria:

Visual design and rhetoric: effectiveness of the visualizations and interactions, based on the usage scenario, intended tasks, and intended audience, incorporation of an innovative view, the usage of rhetorical devices to communicate your story.
Data story: you can think of this project akin to a written essay, in the sense that you are communicating a story, only here you are doing so by combining visualizations + narration.

Project Proposal
You will submit a 2-4 page project proposal describing the high level story (or idea) you want to tell. Structure it around the following:

Title + team members
Briefly summarize the project idea (i.e., the story) you plan to implement
Discuss the dataset(s) you intend to use for your project
For at least one dataset, you must perform a data abstraction (if the dataset contains many attributes, you may limit this to 5-6).
Be sure to include both a colloquial description of the data, and the generic/abstract labels.
At a high level, outline the idioms (visualizations and interactions) you are intending to implement for the project. If desired, you can include a mockup figure outlining what you expect to implement (a pen and paper sketch is fine for this, as long as it is legible).
This report will be submitted on Canvas as a PDF.
